Chapter 41: Palisade

Sie erhalten ein Gitter von Quadraten, von denen einige Zahlen enthalten. Ihr Ziel besteht darin, das Gitter in zusammenhängende Regionen zu zerteilen, die alle die gleiche (vorgegebene) Größe haben, so dass jedes Quadrat, das eine Nummer trägt, zu genau dieser Anzahl an Kanten benachbart ist (dazu gehören auch solche zwischen der Innenseite und der Außenseite des Gitters).

Der Dank für dieses Puzzle geht an Nikoli, die es ‘Five Cells’ nennen. [22].

Palisade ist ein Beitrag von Jonas Kölker zu dieser Sammlung.

[22] https://www.nikoli.co.jp/en/puzzles/five_cells/

41.1 Palisade-Steuerungen

Durch Linksklick setzen Sie eine Kante. Rechtsklick zeigt ‘keine Kante’ an.

Alternativ können die Pfeiltasten einen Tastatur-Cursor bewegen. Abhängig von der Einstellung ‘Cursor mode’ (siehe section 41.3) navigiert der Cursor entweder zwischen den Rasterquadraten oder entlang ihrer Ränder. Im Modus ‘Full-grid’, wird Strg während der Pfeiltasten gedrückt, wird eine Kante gesetzt. Umschalt-Pfeiltaste schaltet eine Kante aus. Im Modus ‘Half-grid’, drücken Sie die Eingabetaste, um eine Kante zu platzieren, und die Leertaste, um eine Kante auszuschalten. In beiden Modi können Sie eine Aktion wiederholen, um ihr Gegenteil auszulösen.

(Alle in section 2.1 beschriebene Aktionen sind auch verfügbar.)

41.2 Palisade-Parameter

Diese Parameter sind von der Option ‘Custom...’ im Menü ‘Type’ verfügbar.

Width (Breite), Height (Höhe)
Größe des Gitters in Quadraten
Region size
Die Größe der Regionen, in die das Gitter zerteilt werden muss.

41.3 Palisade-Benutzereinstellungen

Auf Plattformen, die Benutzereinstellungen unterstützen, können Sie mit der Option ‘Einstellungen’ im Menü ‘Spiel’ das Verhalten der Cursortasten so konfigurieren, dass entweder zwischen vollständigen Rasterquadraten oder entlang der Ränder der Rasterquadrate navigiert wird.
